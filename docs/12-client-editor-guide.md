# Client editor guide

This starter uses Sanity as the CMS target. Clients should edit structured content fields, not page layout or source code.

## Editable content areas

Common editable fields include:

- Business name, tagline, description, phone, email, address, service area, hours, and social links
- Page titles, SEO titles, SEO descriptions, approved summaries, and calls to action
- Services, menu sections, menu items, products, FAQs, team members, testimonials, and gallery images
- Booking, ordering, ecommerce, giving, livestream, event, map, and social URLs
- Announcement banners and footer/navigation links

## Approval rules

Before production launch, confirm approval for:

- All business facts
- Logo and brand assets
- Photos, videos, PDFs, reviews, and testimonials
- Prices, menus, services, service times, event data, and ministry information
- Ordering, booking, giving, livestream, ecommerce, and form provider URLs

## Editing workflow

1. Log in to Sanity Studio using the agency-approved account.
2. Edit the relevant document type.
3. Keep titles clear and short.
4. Add useful image alt text for every image.
5. Do not paste unapproved reviews, copied website text, stock photos, or sensitive/private information.
6. Save the draft and ask the agency to review if the change affects claims, pricing, legal wording, payment links, giving links, or launch-critical information.

## What clients should not edit directly

- Code files
- Layout components
- Cloudflare settings
- DNS records
- Redirect rules
- Payment, booking, ordering, giving, or ecommerce platform settings unless trained and authorized
- API keys, tokens, webhook secrets, or private credentials
